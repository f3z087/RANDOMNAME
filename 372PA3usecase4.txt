New feature 4 use case: 

Name: Using a GUI with warehouse operations 
Primary actor: Warehouse operator 
Preconditions: 
	1. The software implements a GUI that pops up for the warehouse operator.
	2. The GUI has buttons and operations. 
	3. The GUI is easy to read. 
	4. The warehouse operator can point and click on the operations. 
	5. Each button on the GUI performs the correct action when clicked. 
Success conditions: 
	1. All chosen operations on the GUI performed correctly.
	2. The software executed with no errors.
Main success scenario:  
	1. The warehouse operator runs the application.
	2. The GUI pops up with all operations for the warehouse operator.
	3. The software performs the correct operations from the warehouse operatorâ€™s choice.  
	4. The warehouse operator provides accurate data if prompted by the GUI. 
	5. The operations from the GUI performed successfully.
Alternate scenario(s):  
	1. IF the warehouse operator entered invalid inputs for the GUI: 
		a. Display an error message. 
		b. Prompt warehouse operator to enter valid inputs. 
	2. IF the warehouse operator accidently closes out of the GUI: 
		a. The warehouse operator will run the software again. 
		b. The warehouse operator will complete their desired intentions with the GUI.
